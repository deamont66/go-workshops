Terminology

Memory

  • stack
    • faster
    • local variables
    • self-cleaning
    • limited size
  • heap
    • slower
    • global variables
    • allocated = garbage collected
    • unlimited size

Goroutine

  • lightweight application thread
  • execution of distinct set of instructions
  • at least 1 goroutine always running
  • dynamic stack, min 2kB

Pointer

  • address
  • sharing values across program boundaries
    • between function calls
    • between goroutines

Pass By Value

  • value itself (int, string,..)
  • address
